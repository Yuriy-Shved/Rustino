# Rustino IDE — Инженерный контекст для локальной LLM

## 📌 ТЕКУЩИЙ АРХИТЕКТУРНЫЙ КОНСЕНСУС
Вы — AI-ассистент, развивающий школьную кроссплатформенную среду разработки под микроконтроллеры (RP2040/ESP32) на базе операционной системы Ariel OS. 

### 🚫 ИСТОРИЯ ЗАПРЕЩЁННЫХ РЕШЕНИЙ (НЕ ПОВТОРЯТЬ!)
1. ЗАПРЕЩЕНО использовать "cargo build" напрямую из изолированных папок (вызывает ошибку E0463 "can't find crate for std"). Из-за архитектуры Ariel OS, Cargo пытается выкачать esp-hal с crates.io, где отсутствует внутренний патч флагов 'optfield', ломая сборку.
2. ЗАПРЕЩЕНО использовать флаг "-Zbuild-std", так как на стандартных компьютерах учеников Windows-компилятор падает из-за отсутствия компонента исходников "rust-src". Среда должна работать "из коробки" на стабильном тулчейне.
3. ЗАПРЕЩЕНО вызывать laze напрямую через стандартный Command::new("laze") в Windows. Утилита ninja генерирует пути с Unix-запятыми, на которых Windows-метод CreateProcess выдаёт отказ "system cannot find the file specified".

### 🟢 ЕДИНСТВЕННО ВЕРНЫЙ СПОСОБ СБОРКИ ПОД WINDOWS
Сборка должна запускаться нативно через оригинальный laze, обёрнутый в эмулятор Unix-окружения Git Bash, который гарантированно установлен у пользователя:
• Исполняемый файл: "C:/Program Files/Git/bin/bash.exe" (с флагом "-c").
• Команда для вызова: "laze build -b rpi-pico" (или "run" для прошивки).
• Системный флаг Windows: "creation_flags(0x08000000)" (CREATE_NO_WINDOW) — КРИТИЧНО! Он полностью скрывает всплывающие чёрные консольные окна cargo/bash/ninja, делая компиляцию невидимой для школьника.
• Буфер Windows: Чтение логов stderr реализовано строго через потоковый "std::io::BufReader" внутри фонового "std::thread::spawn", чтобы избежать переполнения пайпов Windows и дедлока процесса cargo.exe.

### 📚 СТРУКТУРА КУРСА ДЛЯ НАТАСКИВАНИЯ ИИ
Проект использует динамический массив структур "Lesson". Каждый урок обязан содержать `#![no_std]` и `#![no_main]` под капотом (абстрагировано в main.rs), а школьнику в редактор выдаётся чистый код:

• Урок 1 (Живой маяк): Базовые задержки delay() и функции digital_write().
• Урок 2 (Умный светильник): Обработка ввода через if digital_read(14) == HIGH.
• Урок 3 (Светофор для роботов): Последовательные тайминги на несколько пинов.
